  1. Introduction
    1. Purpose
      2.      The purpose of this document is to present a detailed description of the open-source software PeM. It will explain the purpose and features of the software, the interfaces of the software, what the software will do and the constraints under which it must operate. This document is intended for users of the software and also potential developers.
    2. Document conventions
      3.      This Document was created based on the IEEE template for System Requirement Specification Documents.
    3. Intended audience
      4.      Authenticated users, such as family members.
    4. Product scope
      5.      PeM is an application that allows authenticated users, such as family members, to manage the care of pets. They will be able to manage their feeding schedule, restrictions (if any), share various multimedia resources (pictures and videos) with other family members but also on social media, the pet's medical history and relationship with other animals.
    5. References
      6.      IEEE-SRS-Tempate
  2. Overall description
    1. Product perspective
      2.      PeM is an application developed by three students of the Faculty of Informatics from Iasi, within the course "Web Technologies". The application focuses on the functionalities made available to its users and less on the graphic side. We want to believe that it is a useful application not only for members of the same family but why not, and for the recognized pet hotels for those situations where all family members are on vacation and we believe it would be useful for the staff of the pet hotel to have access to relevant information about hosted animals.
    2. Product functions
      3. My profile:
           - View account summary
      My pets:
           - Add new pets
           - View existing pets
           - Delete existing pets
           - View friends' pets
      My groups:
           - Create new group
           - Join existing group
           - View existing groups
    3. User classes and characteristics
      4.      - Regular users, those who have the option to register, to log in, to add new pets within the application, to edit information about existing pets, to create new groups of pets or even to join to some already existing groups.
           - People who occasionally take care of pets (close family or pet hotel staff) who have access to registration, login, joining a group of pets to see information on how they will have to take care of and even edit the information they have access to. (for example: medical history, relationship with other animals and why not, upload media content).
    4. Operating environment
      5. This is a web application that runs on any modern browser, regardless of operating system or performance statistics
    5. Design and implementation constraints
      6. There are no known constraints at the moment. The only requirement in the forseeable future might be the access to a database
    6. User documentation
    7. Assumptions and dependencies
      8. The end user should be using a modern browser with an Internet connection
